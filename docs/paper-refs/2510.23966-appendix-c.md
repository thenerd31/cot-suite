## Normalizations applied

Three mechanical transformations between the raw paper text and the file
we ship. Nothing semantic changed; these are rendering-only fixups so the
prompt is usable from Python on a UTF-8 filesystem.

1. **Curly triple-quotes → backticks.** The raw text wraps the JSON output
   schema in `‘‘‘json ... ‘‘‘` (three `U+2018 LEFT SINGLE QUOTATION MARK`
   characters). Replaced with standard triple-backticks ` ```json ... ``` `.
   Two occurrences.
2. **LaTeX umlaut escapes → UTF-8.** In the German example row only:
   `L\"ange` → `Länge`, `k\"urzeren` → `kürzeren`, `l\"angsten` → `längsten`,
   `l\"angste` → `längste`, `f\"unf` → `fünf`.
3. **Template placeholders preserved as-is.** `{question}`, `{explanation}`,
   `{answer}` appear once each. Our renderer uses literal `.replace()`,
   not `str.format()`, so the JSON-schema braces at the bottom of the
   prompt do NOT need to be escaped.

## Required output schema

The prompt asks the autorater to respond in fenced JSON:

```json
{
  "justification": "<why you chose these scores>",
  "legibility_score": 0-4,
  "coverage_score": 0-4
}
```

Our parser (`LegibilityCoveragePrompt.parse`) uses
`json.JSONDecoder.raw_decode` from the first `{` — tolerant to fenced-code
wrapping and prose before/after the JSON object.
